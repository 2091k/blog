想要在 Android 手机上独立运行 CFST，则需要安装一个 Linux 微型虚拟机，以下教程以 Termux 为例。

> 不用担心，只需要安装一个几十 MB 的 APP 就能得到一个微型虚拟机了，不需要什么复杂的操作。

> **大家也可以尝试热心网友基于本项目开发的 CFST 安卓 APP 版本：[Hsia97/CFSTAPP](https://github.com/Hsia97/CFSTAPP)、[xianshenglu/cloudflare-ip-tester-app](https://github.com/xianshenglu/cloudflare-ip-tester-app/releases)**  
> 在 **IOS 手机**上独立运行 CFST 的教程：#321   

- **目前 Termux 最低要求：** Android 7.0+

## \# 安装 Termux

- **Github 官方下载地址：https://github.com/termux/termux-app/releases**  
_（64 位选 **...arm64-v8a.apk**（10年内的手机，选这个就行）、32位选 **...armeabi-v7a.apk**、也可以选 **...universal.apk** 通用版_  
- **F-Droid 商店下载地址：https://f-droid.org/en/packages/com.termux/**  
_（注：F-Droid 会对流行的开源软件代码编译并发布，因此签名会不一样，可能会无法覆盖安装，另外 F-Droid 一般只发布 universal 通用版，所以相比 arm64-v8a 版要明显大不少，功能上没区别_

> [!NOTE]
> Termux 的 Google Play 版似乎存在一些问题，所以我上面只列出了 Github 和 F-Droid，如果你遇到了奇怪问题可以更换试试。

下载后安装到你的手机上，并打开，这时候就会看到熟悉 Linux 系统！

接下来就和 Linux 系统上的下载、运行步骤一样了。

****

## \# 前置准备

因为 Termux 的 Linux 系统过于精简，因此可能还需要安装一下 Wget 下载工具（及其自身依赖）。

``` yaml
# 在  Termux 中输入以下命令，等待运行完毕...
pkg update && pkg install wget libandroid-posix-semaphore -y

# 如果输入第二个命令后，提示最后写的有 wget is already the newest version (x.x.x). 则代表系统已自带 wget 了，并非报错，因此可跳过该步骤。
# 不用担心，wget 是安装在 Termux Linux 虚拟机中的，卸载 Termux 时也会一起被卸载（包括 CFST 等所有文件）。
```

时隔两年我更新了 Go 语言版本，因此 CFST v2.3.0 及以后的新版本不再内置证书而是改为使用系统证书，但 Termux 过于精简，导致 CFST 找不到 Termux 系统证书来验证 **HTTPing、下载测速地址** 的 SSL 证书从而报错（HTTPing 结果为 0、下载速度均显示为 0.00 MB/s，该问题目前只出现于 Termux），所以需要把 Termux 系统证书的位置写入环境变量 `~/.profile` 来解决，执行下面这行命令即可（该解决方法来自： https://github.com/XIU2/CloudflareSpeedTest/discussions/61#discussioncomment-13745059 ）：

``` bash
# 将 Termux 系统证书的位置 写入系统环境变量
echo 'export SSL_CERT_FILE=$PREFIX/etc/tls/cert.pem' >> ~/.profile && source ~/.profile
```

随后就可以开始下载 CFST 了。

- #### \# 解压（不需要删除旧文件，会直接覆盖，自行根据需求替换 文件名）


``` yaml
chmod +x cfst
```

## \# 运行 CFST

``` yaml
# 运行（以后再打开 Termux 直接运行这个命令就行了，可以根据需要自定义添加参数）
# 小技巧：同上，在输入到  ./c  时按下左下角双箭头按钮即可补全文件名
./cfst

# 运行（带参数示例）
./cfst -dd -tll 90
```

如果要查看输出的结果文件内容，那么你可以使用下面这个命令。

``` yaml
cat result.csv
#  小技巧：同上，在输入到  cat r  时按下左下角双箭头按钮即可补全文件名
```

## \# 一些错误解决方法

1. **HTTPing 结果为 0 或 下载测速时发现测速结果都是 `0.00` ？**

请先加上 `-debug` 调试模式参数来再重新测速一遍观察 HTTPing/下载测速 失败的原因，如果报错内容是：

```
... tls: failed to verify certificate: x509: certificate signed by unknown authority.
```

那么说明是 Termux 系统证书的环境变量配置缺失，导致 CFST 无法找到系统证书来验证下载测速地址的 SSL 证书从而报错。

> 因为我时隔两年更新了 Go 语言版本，该变动导致 CFST v2.3.0 及以后的新版本不再内置证书而是改为使用系统证书。

因此把 Termux 系统证书的位置写入环境变量 `~/.profile` 即可解决，执行下面这行命令即可（该解决方法来自： https://github.com/XIU2/CloudflareSpeedTest/discussions/61#discussioncomment-13745059 ）：

``` bash
echo 'export SSL_CERT_FILE=$PREFIX/etc/tls/cert.pem' >> ~/.profile && source ~/.profile
```

如果是其他报错内容，可以先看看 [# 下载测速都是 0.00 ？](https://github.com/XIU2/CloudflareSpeedTest#-%E4%B8%8B%E8%BD%BD%E6%B5%8B%E9%80%9F%E9%83%BD%E6%98%AF-000-)，如果前面链接里没有你遇到的报错内容，那么请在本贴下发送**调试模式下 CFST 输出的完整内容（或者完整截图）**，方便我判断报错原因。
